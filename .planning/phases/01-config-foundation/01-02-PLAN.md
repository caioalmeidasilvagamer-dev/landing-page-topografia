Create `lib/config-helpers.ts` with derived-value utility functions that components will import. These functions take a SiteConfig object and return computed values.

**Functions to implement:**

1. **`getWhatsAppUrl(config: SiteConfig): string`**
   - Takes `config.contact.whatsapp` (digits string like '5511999999999')
   - Takes `config.whatsapp.message` (pre-filled message)
   - Returns `https://wa.me/${number}?text=${encodedMessage}`
   - Strip non-digits from phone number with `.replace(/\D/g, '')`
   - URL-encode the message with `encodeURIComponent()`

2. **`getJsonLd(config: SiteConfig): object`**
   - Returns a JSON-LD structured data object for LocalBusiness schema
   - Fields from config:
     - `@context`: `'https://schema.org'`
     - `@type`: `'LocalBusiness'`
     - `name`: from `config.brand.name`
     - `description`: from `config.seo.description`
     - `telephone`: from `config.contact.phone`
     - `email`: from `config.contact.email`
     - `address`: from `config.contact.address`
     - `url`: will be set by the layout component (not in config)
   - Include `sameAs` array from `config.social.instagram` and `config.social.linkedin` (filter out undefined)
   - Include `aggregateRating` from `config.googleRating` if available

3. **`getNavLinks(config: SiteConfig): Array<{label: string, href: string}>`**
   - Returns the navigation links for the navbar
   - Hardcode the 5 nav links (these are fixed, not configurable via config):
     ```
     { label: 'Serviços', href: '#servicos' }
     { label: 'Diferenciais', href: '#diferenciais' }
     { label: 'Projetos', href: '#projetos' }
     { label: 'Depoimentos', href: '#depoimentos' }

     ```
   - NOTE: These could later be made configurable via a `navLinks` section in config, but for now they match the existing navbar.tsx hardcoded values.

4. **`getServiceNames(config: SiteConfig): string[]`**
   - Returns an array of service title strings from `config.services`

   - Also appends `'Outro'` at the end (matching contact-form.tsx line 38)
